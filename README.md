# mainsite-hth-archive

本仓库用于归档和发布多个独立的 HTML 页面。这些页面不隶属于任何特定网站或域名，仅作为静态资源的集中存储与展示。

## 项目简介

mainsite-hth-archive 是一个轻量级的页面归档仓库。仓库中存放的 HTML 文件均为独立页面，彼此之间无依赖关系。适合用于保存历史快照、示例页面、原型设计或临时发布的静态内容。

## 目录说明

```
.
├── index.html          # 仓库入口页面（如有）
├── pages/              # 归档页面存放目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有归档的 HTML 页面文件。每个文件代表一个独立的归档条目。
- `assets/`：存放页面可能引用的公共资源文件。资源文件按类型分目录组织。
- 根目录下的 `index.html` 为可选的导航或索引页面。

## 页面归档说明

### 归档内容

- 所有 HTML 页面均为独立文件，可单独访问。
- 页面可能包含内联样式、脚本，或引用 `assets/` 中的公共资源。
- 页面内容不针对特定域名或网站，不包含商业推广或诱导点击信息。

### 归档原则

- 每个页面应有明确的主题或用途说明（可在文件内注释或通过文件名体现）。
- 页面之间不共享状态或数据。
- 不保证所有页面均能正常渲染或功能完整——部分页面可能仅为静态快照。

## 维护说明

- 本仓库的维护者会定期审查归档内容，可能移除过时或无效的页面。
- 提交新归档页面时，请遵循以下基本规范：
  - 文件命名使用小写字母、数字和连字符（如 `my-page.html`）。
  - 如有引用外部资源，优先使用相对路径指向 `assets/` 目录。
  - 避免在页面中包含敏感信息或动态脚本（如涉及用户输入、数据提交等）。
- 如需删除或更新归档页面，请直接发起 Pull Request 或联系维护者。

## 使用方式

你可以通过以下方式使用本仓库的内容：

1. **直接浏览**：通过 GitHub Pages 或直接访问仓库中的 HTML 文件（需启用静态页面服务）。
2. **下载到本地**：克隆或下载仓库后，在本地打开 HTML 文件即可查看。
3. **引用资源**：如果需要在其他项目中使用 `assets/` 中的资源文件，请确保遵守相关文件的许可协议（如有）。
